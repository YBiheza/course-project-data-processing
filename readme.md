Пояснительная записка к курсовой работе по дисциплине 
“SQL and Data Processing”

Тема: музыкальная школа: статистика и успеваемость студентов.

    OLTP - здесь мы храним:
departments - названия отделений
instruments - названия инструментов и к каким отделениям они относятся
ellectives - название эллективов - дополнительных предметов
subjects - информация о предметах, изучаемых в музыкальной школе
students - данные о студентах, отделениях, где они обучаются, и инструментах, на которых они играют, и эллективах, которые они посещают.
teachers - информация об учителях, инструментах, на которых они специализируются, и предметах, которые они ведут.
grades - информация об отметках учеников по разным дисциплинам, поделенная по семестрам
student-teacher - таблица для связи “много ко многим”, реализует связь какой студент какую дисциплину изучает у какого преподавателя.
    OLAP - здесь мы храним:
Справочники:
DimStudent - информация о студентах (личные данные, данные об обучении в школе, реализация SCD type 2)
DimSubjects - предметы музыкальной школы
DimDepartment - информация об отделениях музыкальной школы
DimInstrument - информация о том, какие инструменты есть в муз. школе и на каких отделениях
DimTeacher - информация об учителях муз. школы.
BridgeTable - таблица связей “ученик-предмет-учитель”
Таблицы фактов
FactGrade - отметки для разных “bridge” (разных учеников по разным предметам) в разные семестры
FactPerformance - средний балл разных учеников в разные семестры 
OLAP database context:
Мы можем получить из OLAP статистическую информацию о музыкальной школе (кто на каком отделении и кто на каком инструменте играет, с какими учителями занимается), а также информация об успеваемости студентов (разделенную по ученикам, преподавателям и отделениям).

    Overall description:
oltp schema - схема OLTP, где все связи сделаны foreign keys, есть отметки PK и unique.
create_oltp.sql - скрипт для создания таблиц в OLTP. Cделаны ограничения unique для отслеживания и определения поведения при конфликтах. Ключ типа STхх является бизнес-ключом, уникально идентифицирующий студента.
insert_into_oltp.sql - скрипт для наполнения таблиц из .csv файлов. В скрипте есть более подробные комментарии. Скрипт является перезапускаемым, при этом записанные данные не перезаписываются (реализовано за счет on conflict do update set …, where … is distinct from …)
postgres fdw.sql - скрипт для связи БД между собой
create_olap.sql - скрипт для создания таблиц в OLAP. Реализованы поля date_from, date_to, isCurrent для реализации SCD type 2. Есть Bridge table для связи ученик-учитель-дисциплина.  В скрипте есть более подробные комментарии. 
insert_into_oltp.sql - скрипт для наполнения таблиц из oltp. Записываются новые данные, без перезаписи старых
olap schema.drawio.png - snowflake schema реализации DWH (OLAP)
oltp query 1-3.sql - скрипты для запуска на базе OLTP, показывают информацию по запросу из OLTP.
olap query 1-3.sql - скрипты для запуска на базе OLAP, показывают информацию по запросу из OLAP

/*================================================*/
    How to run:
Создайте на сервере базу данных (назовем: Music_School)
запустите скрипт creating_oltp.sql
откройте скрипт insert_into_oltp.sql и пропишите там свои абсолютные пути к файлам csv, сохраните 
запустите файл insert_into_oltp.sql
создайте базу данных (назовем MusicSchool_DWH)
запустите скрипт create_olap.sql
откройте скрипт postgres fdw.sql, вставьте свои данные, сохраните, запустите
запустите файл insert_into_olap.sql
чтобы проверить запросы к OLTP и OLAP, запускайте поочередно файлы oltp query 1-3 на базе Music_School (OLTP) и olap query 1-3 на базе MusicSchool_DWH (OLAP).

    Power BI report 
Отчет содержит комментарии и подписи. Первая страница показывает статистику по музыкальной школе по количеству студентов.
Вторая страница - аналог школьного журнала. Видны отметки и средние баллы, можно установить фильтр по семестрам.
Третья страница - аналитика успеваемости по разным группам или по преподавателям.
